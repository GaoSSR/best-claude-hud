# Logo Generation Prompt

Use this prompt to generate the final `best-claude-hud` logo.

## Primary Prompt

Create a clean, modern open source project logo for a Rust CLI called "best-claude-hud". The product is a minimal Claude Code terminal statusline HUD that shows model, git branch, and context window usage. The logo should feel like a premium developer tool: compact terminal HUD, subtle command-line interface, calm technical confidence, high readability, not cute, not noisy, not corporate stock art. Use a dark terminal panel with a single elegant statusline containing tiny segments: "model | git | ctx 19%". Add a small abstract HUD mark or cursor shape that can work as an app icon. Visual style: crisp vector-like illustration, high contrast, polished open-source README hero asset, transparent or very light background, blue/cyan accent with small green status cue, restrained colors, no gradients dominating the whole image. Include the text "best-claude-hud" as a separate wordmark version. No animals, no mascots, no human characters, no clutter, no screenshots, no fake UI text except the tiny statusline segments.

## Output Requirements

- Square icon: 1024x1024 PNG, transparent background.
- README hero/logo: 1600x900 PNG, transparent or white background.
- Wordmark: horizontal transparent PNG and SVG-like clean lettering if available.
- Keep the design legible at 64px.
- Avoid small details that disappear in GitHub README rendering.

## Suggested Asset Names

- `assets/best-claude-hud-logo.png`
- `assets/best-claude-hud-logo@2x.png`
- `assets/best-claude-hud-wordmark-light.svg`
- `assets/best-claude-hud-wordmark-dark.svg`

## Negative Prompt

No animals, no mascot, no robot character, no 3D render, no neon cyberpunk overload, no complex dashboard, no busy background, no random code blocks, no unreadable tiny text, no Claude/Anthropic trademark imitation, no GitHub logo, no terminal screenshot, no photorealism.
